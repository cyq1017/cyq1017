## 文渊使用指南

### G-1. 每日工作流描述
- [ ] start-my-day 补充：会交互问你问题 + 推送学术动态 + 生成日记
- [ ] 工作中的 skill 列表补全（不只有 /read 和 /write）

### G-2. 日记模板
- [ ] "灵感和总结"改为也包含"项目进展"

> [!note] 讨论
> **Q:** 能不能同时保留，或者说有没有同时保留的必要？
> **A:** 可以同时保留。日记模板改为："阅读与思考"、"项目进展"、"灵感与想法"、"当日总结"。不冲突，各记各的，空的就留空。

### G-3. 50_资源/ 目录设计
- [ ] 参考 orbit 的 digest 机制：start-my-day 推送的内容生成 digest 到 `50_资源/学术动态/YYYY-MM-DD.md`
- [ ] 包含：精选推荐 + 描述 + 角度分析
- [ ] 更新 start-my-day skill 和 academic-feed skill 输出到此目录
- [ ] 在偏好设置中增加"信息订阅"区域，支持学术 + 非学术兴趣

> [!note] 讨论
> **Q:** 学术动态里面包含哪些内容？个人 newsletter 又应该包括哪些？
> **A:** 学术动态：教职/博后、学术会议 CFP、期刊征稿、领域新论文、基金机会。非学术兴趣：用户自定义（AI 工具、数字人文方法等）。不预设内容方向，由用户在偏好设置里定义，首次 start-my-day 会问。每类每次最多 3-5 条。
>
> **Q:** 那这个要在 tutorial 里面告诉用户，让他来设定好。
> **A:** 是的。在 tutorial 的 Step 1 里加一步：首次使用时 AI 会问你关注哪些领域，并写入偏好设置。同时在偏好设置模板里预留"信息订阅"区域。
>
> **Q:** 关注领域是会变动的。
> **A:** 偏好设置就是一个普通 md 文件，随时可以改。也可以直接跟 Claude Code 说"我现在对 XX 也感兴趣了"，它会帮你更新偏好设置。不需要一次定死。
